Key Takeaways

✓Two independent backends (PHP + Node) sharing one frontend
✓OC5: half the size, half the deps, half the deploy steps
✓Shared oc-frontend — every JS fix benefits both stacks
✓Template pipeline: OC4 Twig → convert.sh → OC5 Nunjucks
✓Templates are canonical in OC4, derived in OC5 — never diverged
✓Deterministic deploy: one command from VM to running stack

The database is the asset. The rest is replaceable.
